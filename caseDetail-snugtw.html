新思科技- 線上研討會 SNUG Taiwan

實際網站 website
  1. 專案背景 Project Overview

    2019年協助新思科技建立新思線上研討會前台及後台CMS，當時的前台網站主要功能僅是實體研討會資訊公告、報告獲獎名單，以及發布實體研討會影片檔供會員觀看等等。後台CMS則是管理前台頁面呈現的內容以及會員註冊審核等較基本的功能。

    2020年客戶提出虛擬會館概念需求，後來又因為covid-19疫情關係，而更加確定需增加的功能項目，比如線上實況轉播、限定會員觀看、觀看指定影片即可參加抽獎、觀看指定數量影片即可獲得贈品、填寫問卷抽獎……等等，欲取代實體研討會，讓活動可在線上正常舉辦。

  2. 我的角色 My role
    UI/UX Designer Project Manager
  3. 負責項目 Deliverables
    • 專案管理，負責與客戶說明討論介面流程和功能細項，並與工程師描述客戶需求詢問實作可行性(ppt+adobe XD)
    • 2019年前台頁面設計，2020年改版
    • 後台CMS介面流程設計(15項主功能，14項次功能)
    • 使用bootstrap/html/css/jQuery，切版前後台頁面(RWD)
    • 撰寫前後台操作手冊(含合作影音串流商相關操作)，共35頁(.word)
    • 撰寫直播影片操作手冊，共18頁(.word)
    • 收集客戶問題，整理待修改項目並追蹤修改進度(.excel)
    • 功能測試、查找重現bug並整理成待修改列表(.excel)
  4. 設計目標 Design Goal
    • 【前台網站】
      讓User能於各裝置使用前台網站順利註冊參與線上研討會、即時觀看實況影片、填寫問卷或觀看指定影片即可參加抽獎等等。
    • 【後台CMS*2：一個平台，一個後台】
      讓網站管理人員可任意新增研討會網站，個別研討會網站也可分別管理會員、網站內容、影片、匯出名單進行抽獎等等。
  5. 過程與挑戰 Process
    • 一開始在設計後台頁面流程的想法其實和原本公司既有專案的流程架構有點不同，這讓工程師起初有點無所適從，後來我瀏覽公司其他既有專案的後台以及和工程師討論想法後，便有了共識，繼續專心開發如期完成並上線。
    • 第一次擔任PM和客戶討論流程有些緊張，故開會前準備了ppt+介面畫面搭配文字及口頭說明，客戶都有了解到操作，討論過程中客戶也會發現到功能可能有小小的遺漏。比如說在實際使用上其實會需要更多不同的條件過濾搜尋結果。因為有了介面討論，而更可提早發現功能或操作上的問題，為後續開發更加順利。
